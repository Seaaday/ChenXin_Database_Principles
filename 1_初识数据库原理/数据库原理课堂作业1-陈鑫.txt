关系型数据库，英文名是relational database，是采用关系模型来组织数据的数据库。以下是对2020年2月数据库流行度排行榜前十的各种关系型数据库系统的初步调研。前十名中有些属于关系型数据库，有些则不属于，因而没有收入本文。本文简要介绍其各自的优点及缺点、使用场景。

1.Oracle Database
Oracle是甲骨文公司的一款关系型数据库管理系统。其系统可移植性好、使用方便、功能强大，优点是安全性强、稳定性强。适用于各类大、中、小、微机环境。是一种高效率、可靠性好、适应高吞吐量的数据库方案。处理并发强，锁机制非常适合对并发性要求很高的应用。

2.MySQL
MySQL数据库是基于Linux操作系统开发出来的数据库，众所周知Linux是开放源码的操作系统，所以MySQL数据库也是开放源码的免费数据库，这也是MySQL数据库的优点，虽然其功能的多样性和性能的稳定性差强人意。它的成本低，且开源，且支持linux平台。

3. Microsoft SQL Server 
Microsoft SQL Server 是Microsoft 公司推出的关系型数据库管理系统。具有使用方便可伸缩性好与相关软件集成程度高等优点，安全性好，可跨越从运行Microsoft Windows 98 的膝上型电脑到运行Microsoft Windows 2012 的大型多处理器的服务器等多种平台使用。但也有一些缺点：开放性低。只能运行在微软的windows平台，没有丝毫的开放性可言。可伸缩性有限。并行实施和共存模型并不成熟，很难处理日益增多的用户数和数据卷。性能稳定性较差。SQLServer当用户连接多时性能会变的很差，并且不够稳定。使用风险较高。SQLServer完全重写的代码，经历了长期的测试，不断延迟，许多功能需时间来证明。并不十分兼容早期产品。使用需要冒一定风险。客户端支持及应用模式较少。只支持C/S模式。
总的来说，它是一个比较全面的数据库平台。

4.PostgreSQL
PostgreSQL是一种特性非常齐全的自由软件的对象-关系型数据库管理系统（ORDBMS），是以加州大学计算机系开发的POSTGRES，4.2版本为基础的对象关系型数据库管理系统。POSTGRES的许多领先概念只是在比较迟的时候才出现在商业网站数据库中。PostgreSQL支持大部分的SQL标准并且提供了很多其他现代特性，如复杂查询、外键、触发器、视图、事务完整性、多版本并发控制等。同样，PostgreSQL也可以用许多方法扩展，例如通过增加新的数据类型、函数、操作符、聚集函数、索引方法、过程语言等。另外，因为许可证的灵活，任何人都可以以任何目的免费使用、修改和分发PostgreSQL。

5.IBM Bb2
DB2是IBM出品的一系列关系型数据库管理系统，分别在不同的操作系统平台上服务。虽然DB2产品是基于UNIX的系统和个人计算机操作系统，但在基于UNIX系统和微软在windows系统下的Access方面，DB2追寻了ORACLE的数据库产品。DB2具有很好的网络支持能力，每个子系统可以连接十几万个分布式用户，可同时激活上千个活动线程，对大型分布式应用系统尤为适用。目前DB2一般用于银行要求稳定的系统中；所以工作时面向金融的一般都会接触到DB2。

6.Microsoft Access
Microsoft Office Access是由微软发布的关系数据库管理系统。它结合了 MicrosoftJet Database Engine 和 图形用户界面两项特点，是 Microsoft Office 的系统程序之一。Microsoft Office Access是微软把数据库引擎的图形用户界面和软件开发工具结合在一起的一个数据库管理系统。它是微软OFFICE的一个成员，以它自己的格式将数据存储在基于Access Jet的数据库引擎里。它还可以直接导入或者链接数据(这些数据存储在其他应用程序和数据库)。它提供了一个丰富的开发环境，但不能承载过大的数据内容，稳定性、安全性较低。它可以用来进行数据分析： access有强大的数据处理、统计分析能力，利用access的查询功能，可以方便地进行各类汇总、平均等统计。并可灵活设置统计的条件。还可以用来开发软件：比如生产管理、销售管理、库存管理等各类企业管理软件，其最大的优点是：易学！非计算机专业的人员，也能学会。低成本地满足了那些从事企业管理工作的人员的管理需要，通过软件来规范同事、下属的行为

7.SQLite
SQLite是一款轻型的数据库，是遵守ACID的关系型数据库管理系统，它包含在一个相对小的C库中。它是D.RichardHipp建立的公有领域项目。它的设计目标是嵌入式的，而且目前已经在很多嵌入式产品中使用了它，它占用资源非常的低，在嵌入式设备中，可能只需要几百K的内存就够了，因此在电脑，手机，相机，家用电子设备等设备的嵌入式软件是非常好的选择。SQLite非常容易学习和使用。它不需要任何安装和配置，可以用作临时数据集，以对应用程序中的一些数据进行一些处理。只需复制计算机中的SQLite库，就可以创建数据库了。但是SQLite对于高并发高流量的应用不适用。

以上是对目前主流的关系型数据库做的小调研，比较浅显，以后希望能够通过不断的学习和实践操作加深对各种数据库的了解。
